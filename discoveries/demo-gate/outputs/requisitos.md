# Requisitos Candidatos — demo-gate

---

- **[R-01]** El sistema de facturación debe completar el proceso de cobro sin errores durante una transacción activa con el cliente presente.
  - Tipo: funcional
  - Origen: `entrevista-vendedor-real.md` · V. (vendedor)

- **[R-02]** El sistema debe mostrar un mensaje de error comprensible (con causa y acción sugerida) cuando falle el proceso de cobro.
  - Tipo: funcional
  - Origen: `entrevista-vendedor-real.md` · V. (vendedor)

- **[R-03]** El sistema debe ofrecer un canal de soporte inmediato (o un procedimiento de emergencia) accesible durante una situación de venta activa.
  - Tipo: funcional
  - Origen: `entrevista-vendedor-real.md` · V. (vendedor)

- **[R-04]** El sistema de facturación debe estar disponible con una tasa de fallo inferior a la actual (al menos dos o tres fallos semanales registrados) durante horario de ventas.
  - Tipo: no funcional
  - Origen: `entrevista-vendedor-real.md` · V. (vendedor)

- **[R-05]** El tiempo de respuesta del soporte técnico ante un fallo en cobro no debe superar los 5 minutos cuando hay una venta activa en curso.
  - Tipo: no funcional
  - Origen: `entrevista-vendedor-real.md` · V. (vendedor)
